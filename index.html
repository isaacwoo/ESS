📝 はじめに

💬

AIの進化のスピードは、誰もが肌で感じているはずです。少し前まで、「AIに対してどう質問すればいいか」が最重要スキルでした。しかし今、その時代はすでに静かに幕を閉じています。AIを取り巻く新技術・新コンセプト・新用語が、かつてないスピードで次々と登場し、業界の境界は数ヶ月ごとに塗り替えられています。

このような状況の中で、ポジションや担当分野を問わず、積極的に学び、時代のスピードに追いつくことは、もはや"プラスα"ではなく"必須回答" になっています。流行を追いかけるためではなく、この波に乗り遅れないために。

本ノートでは、AIの世界に最近登場した重要な概念や新興技術を体系的に整理しています。細部の全てを深掘りする必要はありません。まずは基本的な認識の枠組みを作り、これらの言葉を見たときに「知ってる」と感じられるようになる——そこを目標としています。

本資料が、AIの新しい世界を探索する 入口 となれば幸いです。

📊 全体像：対応表

まず全体を俯瞰するために、「自分が会社を立ち上げた」 という比喩で考えてみましょう。

技術概念 会社の中の役割 ひとことで言うと
大モデル（LLM） 🧠 天才社員 頭脳明晰だが、入社初日
Prompt 🗣️ 口頭での指示 一時的で、言ったら消える
Agent ⚡ 自律的に動く状態 指示待ちから、自分で考えて動く
Skill 📋 SOPマニュアル 再利用可能な専門知識パッケージ
MCP 🔑 入館証（IDカード） 外部システムへの標準アクセスパス
IDE（Cursor / Trae） 🏢 スマートオフィス AI が既に座っているオフィス環境
Claude Code / OpenCode 🪖 特殊部隊・現場直行モード オフィス不要、ターミナルで直接作業

💡 7つの概念を詳しく解説

🧠
大モデル（LLM）= 天才社員
GPT / Claude / DeepSeek / Gemini など
基礎
🏢 比喩：入社したばかりの天才社員

大モデル（LLM：Large Language Model）は、あなたが採用した 天才社員 だと思ってください。

  • 頭脳明晰で、文書作成も、コード修正も、詩作でもこなせる
  • しかし、今日が入社初日——あなたの会社のルールや流儀を知らない
  • 最大の問題はIQではなく、「社内のやり方を知らない」こと
💡
後続の全ての概念（Prompt / Skill / Agent / MCP）は本質的に同じ問題を解決しています：この天才社員を、本当に自社の人間にするにはどうすればいいか。
🗣️
Prompt（プロンプト）= 口頭での指示
一番直接的なAIとのコミュニケーション
基礎
🏢 比喩：デスクに行って直接話しかける

あなたが社員のそばに行って、直接こう言います：

  • 「この企画書を書いて」
  • 「トーンはプロフェッショナルに」
  • 「形容詞を減らして、シンプルに」

この「その場で言った言葉」が Prompt です。

✅ 強み

即時性が高い・柔軟に使える

⚠️ 弱み

一時的・翌日には覚えていない

📌
2026年の視点： Promptは「消えた」のではなく、「表舞台から裏方へ」移行しました。 Agent が動くたびに自動生成される Prompt が裏で稼働しています。心臓が見えなくても動いているように。
⚡
Agent（エージェント）= 自律的に動く状態
目標を与えると自分で完遂するモード
重要
🏢 比喩：会議中に自分で調べて仕上げてくる社員

「競合分析レポートを作っておいて」と言って会議に行ったら、2時間後に完成版が届いていた——それが Agent です。

  • 自分でタスクを分解する
  • 自分でステップを計画する
  • 自分でツールを呼び出す
  • 自分で結果を確認して修正する
⚡
以前のAI： あなたが一歩言えば一歩動く（あなたが監督）
Agent モード： 目標を与えたら最後まで自分でやり遂げる（AIが自律）
⚠️
注意： Agent は製品名ではありません。「Agentをダウンロードする」とは言いません。Agentは動作モード（働き方）です。 Manus / Devin / Claude Code など多くのツールがAgent能力を持っています。
📋
Skill（スキル）= SOPマニュアル
再利用可能な専門知識パッケージ
重要
🏢 比喩：会社の業務マニュアル（SOP）

社員の仕事が毎回バラバラにならないよう、標準手順書（SOP） を作って渡したようなものです：

  • 標準フロー：「競合分析の手順はステップ1から4まで…」
  • テンプレート：「資料のフォーマットはこの通り」
  • 参考事例：「過去の優秀な成果物をここに格納」
  • 補助スクリプト：「データ整理はこのスクリプトで一発」

Prompt vs Skill の違い

Prompt

口頭の一時指示。言ったら消える。毎回言い直す必要がある。

Skill

書き下ろしたマニュアル。棚に置けばいつでも参照できる。再利用可能。

🌟
ポイント： Skill のライブラリが充実するほど、AIは「ベテラン社員」に近づきます。あなたが積み上げた Skill が多ければ多いほど、AIの成果物の品質は安定し、効率は少なくとも3倍以上 になります。
🔑
MCP = 入館証（IDカード）
Model Context Protocol｜外部システムへの標準アクセス
技術仕様
🏢 比喩：社員証（マスターキー）

能力も方法もある社員が、社内のシステムに入れない——そのために入館証を発行しました：

  • 🗄️ データベースのドアを開く → 履歴データを照会
  • 👥 CRMのドアを開く → 顧客情報を確認
  • 💻 コードリポジトリのドアを開く → 最新コードを取得
  • 🔍 検索エンジンのドアを開く → リアルタイム情報を調べる
📡
現在のMCP対応サービス（2026年2月時点）：1,000以上
GitHub / Slack / Notion / Figma など主要ツールが対応済み

Skill vs MCP の違い

Skill

「どう仕事するか」を教える → 能力・方法論

MCP

「どこに入れるか」を与える → 権限・アクセス

🔒
Skill がなければ、データがあっても分析できない。MCP がなければ、方法があってもデータが取れない。両方が必要です。
🏢
IDE（Cursor / Trae）= AIが既に座っているオフィス
統合開発環境の進化形
ツール
🏢 比喩：入った瞬間AIが隣にいるスマートオフィス

昔のIDEは「普通のオフィス」——机と椅子はあるが、自分で全部やる。

今のAI IDE は、ドアを開けたらもうAIが隣に座っていて：コードを書けば補完してくれ、バグが出れば分析してくれ、リファクタリングを頼めば修正してくれます。

🏢 比喩：オフィス不要、現場でそのまま作業する特殊部隊

IDEのAIが「オフィスの白カラー社員」なら、Claude Code / OpenCode は「工事現場で直接手を動かす特殊部隊」です。

  • GUIは不要——ターミナル（コマンドライン）だけで動く
  • ファイルシステムを直接読み書き・コマンド実行・スクリプト実行
  • プロジェクト全体の構造を一気に理解できる
  • 複数のAgentを並行で走らせることも可能

Claude Code vs OpenCode

Claude Code

Anthropic公式製品。Claudeモデルのみ対応。iOS（iPhoneのような閉じた体験）

OpenCode

オープンソース。Claude / GPT / DeepSeek / Geminiなど75種以上対応。Android（自由にカスタム）

📌
成果物の品質は大差なし。選び方は「どのモデルに慣れているか」だけで決めて大丈夫です。

⚖️ 混同しやすい概念の比較

🤔 「Skill と Prompt は同じじゃないの？」
Prompt

その場の口頭指示。一回限り。翌日には覚えていない。

Skill

文書化されたマニュアル。棚に置けていつでも参照できる。再利用可能。

一言でいえば：Prompt = 一時指示 / Skill = 再利用可能な専門知識パッケージ

🤔 「MCP と Skill は両方とも AI に機能を追加するんじゃないの？」

よくある誤解です。

  • Skill：「どうやって仕事するか」を教える → 能力・方法論
  • MCP：「どこに入れるか」を与える → 権限・アクセス手段

競合分析を例にすると：Skill は「競合分析の方法論」を提供し、MCP は「競合データが置いてあるデータベースへのアクセス権」を提供します。どちらが欠けても仕事は完成しません。

🤔 「Claude Code と Cursor の IDE の違いは？」
IDE（Cursor など）

グラフィカルなオフィスで仕事。どのファイルを変更したか目で見える。

Claude Code

ターミナルから直接ファイルシステムを操作。GUI不要。複数Agentの並行実行が得意。

どちらも仕事は完成させられます。複雑なシナリオでは、現場直行型の方が「感覚が鋭く、反応が速い」ことがあります。

🚀 実践事例：20分で完成した技術調査レポート

📌
以下は、これらの概念が実際に連携して動いた実例です。全ての要素が噛み合ったとき何が起きるか、感じてください。

タスク： 新プロジェクトのための完全な技術選定レポートを作成（5つのフレームワークを比較）

以前のやり方： 十数個のWebサイトを手動で閲覧、メモ、比較表作成、分析記述、整形 → 最低2〜3日

🪖 Claude Code 起動（特殊部隊が現場へ）
ターミナルを開き、目標を1つ指定：「この5つのフレームワークの技術選定レポートを作って」
⚡ Agent モードに突入（自律的に動き始める）
自分でタスクを分解し、計画を立て、実行開始
🔑 MCP → GitHub 接続（入館証を使ってデータ取得）
各フレームワークの最新コードとStarデータを取得
🔑 MCP → 検索エンジン接続（最新コミュニティ動向も調査）
過去3ヶ月のコミュニティ議論の熱量データを取得
📋 Skill 「技術選定分析」を参照（SOPに従って比較実行）
事前に作成した標準フローに沿って分析を進める
✅ 完成！（所要時間：約20分）
データ表・比較図・結論・推奨事項を含む完全なレポートが手元に
🎯
これは「AIが少し時間を節約してくれた」のではありません。
大モデルの能力 × Agentの自律性 × Skillの方法論 × MCPの接続力——4つの歯車が噛み合ったとき、効率の次元そのものが変わります。

これは「仕事が少し速くなった」ではなく、「仕事のやり方が再定義された」ということです。

✅ まとめ・2026年のAI活用の本質

7つの概念 最終整理

  1. 大モデル（LLM）：全ての土台。頭脳明晰な天才社員だが、初日は会社のことを知らない
  2. Prompt：最もシンプルな指示手段。一時的だが今も裏方として機能している
  3. Agent：目標を与えたら最後まで自律的にやり遂げるモード（製品名ではなく働き方）
  4. Skill：再利用可能なSOPマニュアル。蓄積するほどAIはベテラン化する
  5. MCP：外部システムへの標準アクセス手段。現在1000以上のサービスが対応
  6. IDE（Cursor/Trae）：AIが既に常駐しているスマートな開発環境
  7. Claude Code / OpenCode：GUIなしでターミナルから直接動く、上級者向けAIエージェント
🌊
2026年のAI活用における最大の優位性は、「技術に詳しいこと」ではありません。

これらの概念を使って、自分の働き方を再定義できるかどうか。

全部を完璧に習得する必要はありません。それぞれが「どこに位置し、何の問題を解決するか」を理解した上で、必要なときに正しいツールを、正しい方法で使える——それが今求められることです。

AIの世界は、一つの概念の独演ではなく、交響楽団全体の演奏です。そして、まだ始まったばかりです。
